How it works

No check-ins. No YAML. It just shows up.

Connect once. Every morning Inky reads what actually happened and writes the standup. The information already lives in your tools — Inky just reads it.

01

Connect your tools

Install the GitHub App, point Inky at your Linear workspace, pick a Discord channel. Read-only, least-privilege. Two minutes.

02

Inky reads what shipped

Commits, PRs, reviews, issues, cycles — collected, de-duplicated, identity-resolved. The numbers are computed, never guessed.

03

You get the standup

A grounded daily + weekly standup, per-person and project-wide, with status vs plan. Read it, or ask Inky about it.

Intention vs execution

What you planned, against what you shipped.

Inky reconciles your tracker with your code and tells you where things actually stand — including the work that never got a ticket. It classifies; it never invents a link.

Planned · Shipped
On track

The issue has linked PR activity and it's moving. The calm case — Inky just confirms it.

Planned · Not shipped
At risk

A committed issue with no code behind it yet. Flagged as a heads-up, not an alarm.

Shipped · Not planned
Untracked

Real work shipped with no ticket. Surfaced as signal — flagging the work, never the person.

The honest gap
No guessing

If nothing links a PR to an issue, Inky says so. “I didn't guess” is the feature.
