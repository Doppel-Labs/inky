How it works

It's a little uncanny how little you do.

Async standups still make everyone type their update. Inky doesn't — the work already lives in your tools, so it reads it and writes the standup. You just read it.

01

Say hi, once

Add the GitHub App, connect Linear, drop Inky in a Discord channel. Read-only and least-privilege — about two minutes.

02

Inky reads up

Every commit, PR, review and issue, collected and de-duplicated. It counts the work; it never guesses a number.

03

The standup appears

Per-person and project-wide, with status vs plan — waiting in your channel before the first coffee.

No meeting, no typing — about 4 hours a week back for a team of ten.

Planned vs shipped

What you meant to do, next to what you did.

Inky lines your tracker up against your code and tells you where things really stand — including the work that never got a ticket. It classifies; it never invents a link.

Planned · Shipped
On track

The issue has a PR behind it and it's moving. The happy case — Inky just confirms it.

Planned · Not shipped
At risk

A committed issue with no code yet. A gentle heads-up, never an alarm.

Shipped · Not planned
Untracked

Real work that shipped with no ticket. Surfaced as signal — flagging the work, never the person.

The honest gap
No guessing

If nothing links a PR to an issue, Inky says so. “I didn't guess” is a feature, not a bug.
